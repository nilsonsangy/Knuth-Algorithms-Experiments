How Algorithm T Works

Algorithm T is split into two simple functions. The offline phase builds a swap plan once. The online phase reuses that plan to generate permutations quickly.

Simple Pseudocode

Algorithm T(n)
  offline:
    create the initial permutation [1..n]
    build the list of adjacent swap positions
    return the swap plan

  online(swap_plan):
    create the initial permutation [1..n]
    output the first permutation
    for each swap position in swap_plan:
      swap the adjacent elements
      output the new permutation

Offline Function

This function prepares the swap plan. It finds which adjacent positions must be swapped to move from one permutation to the next. The result is a compact roadmap.

Online Function

This function receives the precomputed plan and applies each swap in order. It does not recompute the logic of the next move; it only follows the plan.

Step by Step

1. Start with the identity permutation.
2. Build the swap plan offline.
3. Reuse that plan online to generate every new permutation.
4. Measure offline and online costs separately.
5. Compare T with P and Improved P.
